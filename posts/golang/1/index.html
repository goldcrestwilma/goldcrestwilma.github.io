Go 언어의 장점
Table of contents
  • 소개
  • 장점

소개

공식 홈페이지 Go 또는 Golang은 오픈 소스 프로그래밍 언어입니다. 정적으로 유형이 지정되고 컴파일 된 기계 코드 바이너리를 생성합니다. 새로운 프로그래밍 언어에는 메모리를 안전하게 사용하고, 객체를 관리하고, 가비지를 수집하고, 동시성과 함께 정적 (또는 엄격한) 타이핑을 제공 할 수있는 도구가 포함되어 있습니다.

2009년 Google의 Rob Pike, Robert Griesemer, Ken Thompson 덕분에 2009 년 세계가 처음으로 Go에 소개되었습니다. Go를 만드는 주요 목표는 다른 프로그래밍 언어의 최고의 기능을 결합하는 것입니다.

최신 생산성과 함께 사용하기 쉬움

정적 타이핑과 함께 높은 수준의 효율성

네트워킹을위한 고급 성능 및 멀티 코어 전력의 완전한 사용

장점

  1. Flexible - It is concise, simple and easy to read.
  2. Concurrency- It allows multiple process running simultaneously and effectively.
  3. Quick Outcome- Its compilation time is very fast.
  4. Library- It provide a rich standard library.
  5. Garbage collection- It is a key feature of go. Go excels in giving a lot of control over memory allocation and has dramatically reduced latency in the most recent versions of the garbage collector.
  6. It validates for the interface and type embedding.

간결한 코드를 사용하면서 생산성이 향상됐고, 컴파일 속도와 실행속도가 빨라 테스트가 쉬웠다. 정적 타입 자바의 경우에는 JVM 위에서 동작하는 언어이다 보니 JVM 환경 세팅과 의존 라이브러리를 패키징하는 과정이 필요하다. 제대로 성능을 내기 위해서는 JVM, 톰캣, DB 풀 등의 파라미터를 튜닝한다. Go의 경우에는 여러 가지 최적화와 패키징 기능을 언어 차원에서 내장하고 있고, 필요한 라이브러리를 모두 모아 하나의 정적 바이너리를 생성하기 때문에 상대적으로 배포가 간단합니다.
